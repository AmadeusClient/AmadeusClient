~/ scripts — 脚本系统参考

基于 GraalJS（GraalVM Polyglot）的 JavaScript 脚本引擎，提供隔离上下文、模块化注册、事件驱动的完整脚本框架。

概述 #

基本概念

folder 存放目录
~/.zen/liquidres/scripts/
所有 .js 文件会自动被扫描加载。
layers 隔离上下文
每个脚本拥有独立的 GraalJS Context，变量/函数互不冲突。
extension 模块化注册
通过 createModule() 创建的模块自动加入 ModuleManager，可在 ClickGUI 中开关、绑键。
sync 热重载
编辑脚本后可用命令重载，自动恢复模块启用状态，无需重启游戏。
event 事件驱动
支持 35+ 种事件（Tick、Motion、Packet、Render…），注册回调即可响应。
globe 免 import
所有常用 Java 类型、事件类、工具类已预绑定，开箱即用。

控制台命令

命令说明
.script load <文件名.js>加载脚本（文件必须在 scripts/ 目录下）
.script reload自动加载新 .js + 重载已有脚本（无参快捷方式）
.script reload <脚本名>热重载指定脚本（不需要 .js 后缀）
.script reloadall热重载所有已加载脚本（已废弃，直接用 reload）
.script unload <脚本名>卸载脚本（自动禁用并移除其所有模块）
.script list列出所有已加载脚本
提示
脚本名 = 文件名去掉 .js。例如 CenterText.js 的脚本名为 CenterText。

createModule() — 创建脚本模块 #

语法

var mod = createModule("模块名", Category.分类, {
    onTick:       function(e) { /* ... */ },
    onMotion:     function(e) { /* ... */ },
    onRender2D:   function(e) { /* ... */ },
    onEnable:     function()  { /* ... */ },
    onDisable:    function()  { /* ... */ },
    // 支持所有 30+ 事件…
});
参数类型说明
namestring模块名称，显示在 ClickGUI 和 HUD 中
catCategory 枚举 分类：COMBAT / MOVEMENT / PLAYER / RENDER / WORLD / MISC
hooksobject事件回调映射，key 为钩子名，value 为处理函数

返回值是一个 ScriptModule 实例，可以直接调用 mod.setKey()、mod.setEnabled() 等方法。

高级用法

绑定快捷键

mod.setKey(295); // GLFW_F6

键码为 GLFW 键码，不是 Java VK_ 码。常用：F6 = 295、RSHIFT = 345。

创建设置项

var speed = new NumberSetting("Speed", 1.0, 0.1, 10.0, 0.1);
var mode  = new ModeSetting("Mode", "Vanilla", ["Vanilla", "Hypixel"]);
mod.registerSetting(speed);
mod.registerSetting(mode);

// 在事件中读取设置
var s = speed.getValue();       // double
var m = mode.getSelected();     // string

直接监听事件（不创建模块）

var listener = new ScriptModule("__listener", Category.MISC);
listener.when(TickEvent, function(e) {
    if (mc.player != null && mc.player.tickCount % 20 === 0) {
        printChat("§720 ticks passed");
    }
});
Client.getInstance().getModuleManager().register(listener);

玩家访问器 __player

由于 Minecraft SRG 混淆，GraalJS 无法直接通过 MCP 命名访问 mc.player.*（如 mc.player.hurtTime）。
__player 是 Java 侧编译的 PlayerAccessor 包装器，绕过混淆让 JS 脚本安全读写玩家状态。

GraalJS 自动将 getXxx() 映射为 obj.xxx、isXxx()（boolean）映射为 obj.xxx，
所以 JS 中使用 属性读写语法（__player.hurtTime），无需调用 getter。

JS 语法类型可写说明
__player.hurtTimenumber（int）只读受击动画剩余 tick（0 = 未受击）
__player.onGroundboolean只读玩家是否在地面
__player.motionXnumber（double）只读X 轴速度（击退时突增）
__player.motionYnumber（double）只读Y 轴速度（击退时向上飙升）
__player.motionZnumber（double）只读Z 轴速度
__player.healthnumber（float）只读当前生命值
__player.sprintingboolean只读是否在疾跑
__player.sneakingboolean只读是否在潜行
__player.jumpFromGround()void—地面跳跃（空中调用无效，静默忽略）
工作原理

__player 是编译到 JAR 中的 PlayerAccessor 静态内部类实例。
它的 getter 方法体内部的 mc.player.hurtTime 等引用在 reobfJar 阶段被自动重映射为 SRG 名，
所以运行时能正确找到混淆后的 Minecraft 字段。JS 侧完全无需关心 SRG 问题。

示例

// 检测受击 → 地面跳跃抵消击退
if (__player.hurtTime > 0 && __player.onGround) {
    __player.jumpFromGround();
}

// 读取速度分量
var speed = Math.sqrt(
    __player.motionX * __player.motionX +
    __player.motionZ * __player.motionZ
);

// 检测状态
if (__player.sprinting && !__player.sneaking) {
    printChat("疾跑中，未潜行");
}

完整示例 #

CenterText.js — 屏幕中央渲染文字

// 创建渲染类模块
var module = createModule("CenterText", Category.RENDER, {
    onRender2D: function(e) {
        var gui = e.guiGraphics();
        var pose = e.poseStack();
        var sw = mc.getWindow().getGuiScaledWidth();
        var sh = mc.getWindow().getGuiScaledHeight();

        var title = "§bLiquidBounce §7× §aScripts";
        var tw = mc.font.width(title);
        var tx = ((sw - tw) / 2) | 0;
        var ty = ((sh / 2 - 30 - 12)) | 0;

        // 安全使用 drawRoundedRect（颜色要加 | 0 转 int）
        drawRoundedRect(pose, ((sw - tw - 40) / 2) | 0, ty, tw + 40, 60, 8, 0x90000000 | 0);

        gui.drawString(mc.font, title, tx, ty, -1, true); // 带阴影
        gui.drawString(mc.font, "脚本系统已就绪", ((sw - mc.font.width("脚本系统已就绪")) / 2) | 0, ty + 16, 0xAAAAAA, false);
    }
});
module.setKey(295); // GLFW_F6

ChatLog.js — 聊天消息日志记录

var mod = createModule("ChatLog", Category.MISC, {
    onChatReceive: function(e) {
        var msg = e.getMessage().getString(); // Component → String
        java.lang.System.out.println("[CHAT] " + msg);
    },
    onChat: function(e) {
        // 阻止发送含 "badword" 的消息
        if (e.getMessage().contains("badword")) {
            e.setCancelled(true);
            printChat("§c消息被拦截！");
        }
    }
});

AutoRespawn.js — 自动旁观/重生

var mod = createModule("AutoRespawn", Category.PLAYER, {
    onTick: function(e) {
        if (mc.player == null || !mc.player.isSpectator()) return;
        // 按名称搜索方块
        var bed = BlockUtil.findBlock(5, function(pos) {
            return mc.level.getBlockState(pos).getBlock().getName().getString().contains("bed");
        });
        if (bed != null) {
            mc.player.connection.send(new ServerboundMovePlayerPacket.Pos(bed.getX(), bed.getY() + 0.5, bed.getZ(), true));
        }
    }
});

注意事项 #

类型陷阱 — 颜色值超出 int 范围

JavaScript 中 0x90000000 的值是 2415919104，超出了 Java int 最大值（2147483647），GraalJS 会当 Double 传入，导致 Java 方法重载解析失败。

修复：用 | 0 强制转成 32 位有符号整数：0x90000000 | 0。

方法重载 — Double 不能转 float

JS 的所有 number 类型都是 Double，Java 的方法重载不会自动将 Double 转 float。涉及 RenderUtil.drawRoundedRect() 等重载方法时，用 | 0 确保整数传入。

热重载提示

.script reload 使用 脚本名（不带 .js），如 .script reload CenterText。
如果脚本之前没加载过（不在 scripts 目录），先用 .script load 加载。
重载会自动恢复该脚本所有模块的启用/禁用状态。

键码说明

客户端使用 GLFW 键码（不是 Java VK_ 代码）。
常用键码：F6 = 295，RSHIFT = 345，LCONTROL = 341，GRAVE = 96。通过 KeyBind 类可查阅完整映射。

设置项注册后自动显示在 ClickGUI

调用 mod.registerSetting(setting) 后，设置会自动出现在 ClickGUI 的该模块设置面板中。支持 Boolean / Number / Mode / MultiSelect / Color 类型。

预绑定覆盖了所有 client.amadeus.* 类

源码中 client.amadeus.* 下的 所有工具类、事件类、管理器、HUD 基类 都已预绑定。
构建后这些类名会被混淆成随机名，但因为预绑定的 Java.type('...') 字符串也会被 ASM 自动重写，
所以 脚本完全不需要关心混淆后的类名，直接用预绑定变量名就行。

如果需要 java.util.Random 这样的标准库类，依然可以用 var Random = Java.type('java.util.Random');，这些不会被混淆。
